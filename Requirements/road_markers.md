# Road Markers

## Overview

RetroRacing replaces visible horizontal/rectangular grid strokes with a road-marker overlay that better matches perspective gameplay. The system renders discontinuous lane marks that scroll with each grid tick and supports a Big Cars visual override.

## Architecture

- Rendering lives in `GameScene+Grid.swift`.
- Cell fill still comes from `theme.gridCellColor()`.
- Line visuals are drawn as overlays and resolved by precedence:
  - **Big Cars enabled** -> vertical-only mode.
  - **Big Cars disabled + Road Style = Simplified Grid** -> vertical-only mode.
  - **Big Cars disabled + Road Style = Detailed Road** -> detailed road mode.
- Overlay state is tracked via `lineOverlayNodes` and cleared/redrawn every grid refresh.

## Assets

The lap marker uses one shared white mask asset:

- `lapStripMask` – full-width checkered lap strip between outer road boundaries

Road dashed lines are rendered procedurally as row-local trapezoid segments from shared road geometry (no per-lane dash masks at runtime).
`lapStripMask` is generated by `Scripts/GenerateRoadDashMasks.swift` and includes `universal`, `watch`, and `tv` variants.
`lapStripMask` is authored with a wider source aspect ratio so runtime placement across road width preserves intended perspective.
Lap strip geometry uses a strong-but-balanced taper (`~1.2%` bottom inset, `~4.5%` top inset per side) so side inclination matches tightened road-edge convergence without over-tilting.
Lap strip outline is padded inwards and vertically inset to avoid clipped bottom-corner pixels.
The generator writes flat fallback PNGs to `RetroRacingShared/Resources/Sprites`; `lapStripMask.png` is used by the current runtime path, while lane-mask outputs are kept for compatibility.

## Rendering Rules

### Detailed Road (`Big Cars Off` + `Road Style = Detailed Road`)

- Horizontal grid lines are hidden.
- Dashed road lines are built from a shared road-boundary function and rendered as four per-row trapezoid segments.
- Per visible row:
  - outer-left boundary
  - inner-left lane boundary
  - inner-right lane boundary
  - outer-right boundary
  - boundary positions are equally spaced from the same road span, so lane widths remain consistent at each depth
  - line thickness is computed from depth at the row segment top and bottom, preserving continuity between adjacent rows
- Lap markers are rendered only during the two-row safety empty window before a speed increase:
  - rendered as one strip centered between the two tracked safety rows
  - rendered as one full-width strip between road outer boundaries (single sprite)
  - strip height scales by depth using the average safety-row index:
    - `stripHeight = 0.42 * cellHeight * ((averageRowIndex + 1) / 5)`
  - strip contains two perspective checker rows
  - trapezoid outline is baked into the `lapStripMask` asset (no separate runtime outline node)
  - dashed road markers are suppressed on lap-marker rows to avoid visual overlap

### Vertical-Only (`Big Cars On` OR `Road Style = Simplified Grid`)

- `Big Cars On`:
  - Perspective road/lap markers are hidden.
  - Two vertical lane separators are rendered as flat, non-perspective dashed segments (4 rows visible, 1 row empty via `roadDashPhase` cadence).
  - Separator tint uses the same `roadLineColor()` path as detailed mode.
  - Horizontal separators stay hidden.
- `Road Style = Simplified Grid` (`Big Cars Off`):
  - Perspective road/lap markers are hidden.
  - Two continuous vertical separators are rendered.
  - Horizontal separators stay hidden.

## Dash Cadence

- Grid is `5x3`.
- Dash phase state: `roadDashPhase` (`0...4`).
- On each tick update (`.update` / `.updateWithEmptyRow`), `roadDashPhase` increments by 1 modulo 5.
- Lane move actions do not change `roadDashPhase`.
- Empty dash row index: `(roadDashPhase + 4) % 5` (4 rows visible, 1 row empty).

## Perspective and Placement

- Road span is computed from scene Y using a single perspective model:
  - top width ratio: `0.38`
  - bottom width ratio: `0.94`
  - lane boundaries are the 0/1/2/3 thirds of that span
- Dash segment height and lane-line thickness are depth-interpolated from top to bottom so perspective reads smoothly without per-row discontinuities:
  - line width factor: `0.043` (top) -> `0.078` (bottom)
- Lap strip checker columns converge toward the center, with stronger convergence near the outer edges and gentler convergence near the middle.
- Lap strip horizontal bounds use no interior inset and stronger outward expansion from lane-line thickness so the strip reaches the outer side of the outer lane lines.
- Car/lane alignment uses the same road model:
  - lane center X is derived from row depth and lane index
  - player, rival, and crash sprites anchor to those lane centers to keep cars visually centered in their lanes
  - no baseline side-lane convergence offset is applied; optional convergence is additive-only when explicitly enabled
  - rival/crash scale uses a blended depth curve (top row floor `0.31`, linear blend `0.45`, quadratic exponent `2.0`) so the top row stays small while row-to-row separation remains visible near the horizon
- Big Cars sizing:
  - player, rival, and crash all use the same fixed cell-fill box (`0.9`) with lane-centered placement and no perspective scaling, so Dynamic Type users get uniform, high-legibility targets.

## Lap Marker Timing

- Safety-row markers are tracked in scene state (`safetyMarkerRows`, max 2).
- On each tick update:
  - existing safety rows shift down with grid movement
  - one off-screen bottom sentinel row is retained so the lap strip exits smoothly
  - rows are dropped only after moving beyond that sentinel
  - `.updateWithEmptyRow` inserts a new safety marker at row `0`
- Lap markers are rendered only when exactly two tracked safety rows are present, centered halfway between those rows (including top/bottom boundary transitions with one virtual row outside the visible grid).

## Contrast Requirement

- Line tint color (including lap strip tint) comes from `theme.roadLineColor(isIncreaseContrastEnabled:)`.
- When system Increase Contrast is enabled, themes switch to dedicated high-contrast line colors.
- Current high-contrast values for default themes are tuned to clear approximately `>= 4.5:1` against their road fills:
  - Pocket: `#464E28` on `#AFC566`
  - LCD: `#6E6A5F` on `#F5EBD2`

## Testing Strategy

- Scene rendering tests verify:
  - Dash phase cycling across 5 ticks
  - Lane moves do not advance dash phase
  - Big Cars mode overrides road style and uses vertical-only separators
  - Big Cars mode uses flat dashed separators (non-perspective), while Simplified Grid uses continuous vertical separators
  - Simplified Grid mode uses vertical-only separators
  - Detailed Road mode shows dashed road markers
  - Lap markers appear only in the two-row safety window as a single strip centered between the two tracked safety rows
  - Lap strip remains continuous as it approaches/passes the player row and only disappears once off-screen
  - Horizontal grid lines remain hidden
  - Top dashed rows are more converged than bottom rows
  - Bottom inner dash spacing remains tighter than one full lane width
  - Outer dashed line drift is larger than inner dashed line drift
  - Cars align to computed lane centers per row
  - Top-row side rivals are closer to center than lower-row side rivals
- Contrast tests verify:
  - Resolver output always reaches `>= 4.5:1`
  - LCD and Pocket road fills both pass
